    What changed in V5

    V5 focuses on editorial maturity rather than raw expansion. The repository now behaves more like a maintained technical-documentation environment and less like a simple collection of lesson files.

    Main improvements¶

    • section landing pages added for faster orientation;
    • editorial and quality-assurance layer introduced;
    • appendices added for repository governance;
    • MkDocs Material configuration refined;
    • custom stylesheet added for improved readability;
    • module openings standardised with at-a-glance guidance and documentation intent;
    • templates strengthened with clearer usage framing;
    • model-answer and teacher-note layers connected more explicitly to review logic.

    Why this matters¶

    A large repository becomes difficult to maintain when style, structure, and linking conventions remain implicit. V5 reduces that risk by making those conventions visible.

    Recommended next step¶

    Use V5 as the stable baseline for: - final content polishing; - selective topic deepening; - repository publication and iteration.
