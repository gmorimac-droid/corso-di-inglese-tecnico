    Repository Quality Playbook¶

    Playbook use note

    A playbook is most useful when applied in sequence. Follow the review order rather than correcting isolated sentences without a framework.

    Purpose¶

    This playbook helps the learner and the reviewer assess whether the final MkDocs repository behaves like a professional technical knowledge base rather than a collection of disconnected assignments.

    Quality pillars¶

    1. Page identity¶

    Each page should have a clear title, a stable function, and a visible scope.

    2. Internal linking¶

    A page should not remain isolated if related glossaries, exercises, or templates exist elsewhere in the repository.

    3. Reuse value¶

    The page should contain phrasing, structure, or reference logic that could be reused in later professional work.

    4. Navigational logic¶

    Section names and hierarchy should help the reader predict where information lives.

    5. Revision maturity¶

    A premium page usually shows evidence of at least one substantial revision pass.

    Final repository questions¶

    • Can the user move from module to glossary to template without confusion?
    • Are repeated concepts explained consistently?
    • Does the site feel cumulative rather than repetitive?
