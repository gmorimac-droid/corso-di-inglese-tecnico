# Document Control Conventions

## Purpose

This appendix defines the repository-level habits that improve traceability, revision clarity, and maintainability.

## Core conventions

### Titles

Page titles should describe documentary function, not merely topic. Prefer *Incident Report Template* over *Incidents*.

### File names

File names should remain stable, descriptive, and lower-risk for future linking. Avoid file names that depend on temporary project phases or dates unless version history requires them.

### Revisions

Where a page undergoes major change, the revision should be described in commit history and, where useful, in a short visible note inside the page.

### Links

When moving or renaming a page, update inbound and outbound links in the same editorial cycle whenever possible.

### Template control

Templates should include enough structure to guide writers, but not so much prewritten text that they encourage careless copy-paste behaviour.

## Reviewer behaviour

Reviewers should assess not only whether a document sounds acceptable, but also whether it can be found, reused, linked, and maintained.
