# Modules

## What This Section Contains

This section contains the main learning path of the course.

The modules move progressively from foundational technical English to more applied engineering communication, including documentation, reporting, maintenance, safety, and publication-oriented writing.

Each module is designed to help the learner:
- understand key technical language patterns;
- apply them in realistic engineering contexts;
- produce structured outputs that can later be reused in the documentation project.

## At a Glance

| Module Group | Main Focus | Typical Output |
|---|---|---|
| Modules 01-04 | foundations, vocabulary, grammar, technical reading | notes, terminology lists, reading annotations |
| Modules 05-08 | components, systems, processes, performance | descriptions, workflows, summaries |
| Modules 09-12 | technical writing, emails, reports, requirements | emails, report sections, requirement statements |
| Modules 13-15 | maintenance, troubleshooting, safety | fault notes, maintenance summaries, safety-related writing |
| Modules 16-18 | sector-specific language and documentation workflow | terminology mapping, structured documentation, MkDocs output |

## How to Use the Modules

The modules can be used in two ways.

### Guided Study
Follow the modules in numerical order if you want a complete and progressive learning path.

This is the recommended option for learners who want to build vocabulary, grammar control, writing discipline, and documentation habits step by step.

### Targeted Consultation
Go directly to a specific module if you need support in one area, such as:
- technical description;
- reporting;
- requirements language;
- maintenance terminology;
- safety communication;
- documentation workflow.

This option is useful for revision, workplace support, or focused practice.

## Recommended Reading Strategy

A practical way to work with each module is:

1. read the overview and objectives;
2. identify the key terminology;
3. review the main language patterns;
4. complete the task or exercise;
5. revise the output before storing or reusing it.

## Suggested Learning Progression

### Modules 01-04 Foundations
These modules build the core technical-English base of the course.

Focus areas:
- engineering vocabulary;
- grammar for technical communication;
- reading technical texts;
- basic documentary awareness.

### Modules 05-08 Description and Explanation
These modules develop the language needed to describe technical objects and operational logic.

Focus areas:
- components and assemblies;
- systems and architectures;
- processes and workflows;
- performance, testing, and evaluation.

### Modules 09â-12 Documentary Writing
These modules strengthen structured professional writing.

Focus areas:
- technical writing principles;
- technical emails;
- reports and summaries;
- requirements and compliance language.

### Modules 13-15 Operational Communication
These modules focus on communication in maintenance and risk-related contexts.

Focus areas:
- maintenance language;
- troubleshooting and fault analysis;
- safety and incident communication.

### Modules 16-18 Sector and Publication Layer
These modules extend the course toward specialised and publication-oriented output.

Focus areas:
- defence-oriented and institutional technical language;
- programme and procurement communication;
- documentation architecture with MkDocs and GitHub.

## Before You Move to the Next Module

Before progressing, check whether you can:

- understand the core terminology;
- explain the main concepts in simple technical English;
- complete the main task with reasonable control;
- revise your output for clarity and structure.
