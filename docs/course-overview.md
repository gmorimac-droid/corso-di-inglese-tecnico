# Course Overview

## What This Course Is

**Technical English for Engineering B2** is a documentation-focused course for learners who need to use English in engineering environments.

The course supports communication in contexts such as:
- technical reporting;
- documentation and knowledge transfer;
- coordination between teams;
- maintenance and troubleshooting;
- safety and compliance communication;
- project and programme support.

The main goal is not only to understand technical English, but also to produce it in a clear, structured, and professionally usable form.

## Why This Course Matters

In engineering contexts, language is not only a medium of explanation. It is often part of the work itself.

Technical English is used in:
- requirements;
- procedures;
- reports;
- meeting records;
- maintenance logs;
- safety notes;
- documentation pages.

For this reason, the course combines language learning with documentation practice.

## Domain Orientation

The course is broad enough for different engineering environments, but it is especially suitable for:

- mechanical and industrial engineering;
- electrical, electronic, and control-system contexts;
- maintenance and support activities;
- system integration and testing;
- safety, quality, and compliance communication;
- dual-use and defence-oriented industrial settings.

The course does **not** aim to teach military doctrine. It focuses on **neutral, professional, technically controlled English** that can be used in demanding environments where clarity, reliability, and traceability matter.

## Core Principles

The course is built around five practical principles.

### 1. Language must support action
Technical language should help the reader understand, verify, decide, or act.

### 2. Vocabulary must be reusable
Terms are not learned in isolation. They are reused across modules, exercises, templates, and documentation outputs.

### 3. Grammar must support precision
Grammar is treated as a professional tool for expressing obligation, sequence, limitation, causality, and neutral reporting.

### 4. Writing must be document-ready
The course trains the learner to produce texts that can be reviewed, stored, shared, and reused.

### 5. Revision is part of competence
A first draft is rarely enough. The learner is expected to improve wording, structure, consistency, and clarity.

## Course Structure

The course is organised into six learning areas:

1. **Core Technical English**
2. **Engineering Communication**
3. **Documentation, Reporting and Technical Writing**
4. **Operational, Maintenance and Safety English**
5. **Defence, Dual-Use and Institutional Language**
6. **Digital Documentation Project with MkDocs and GitHub**

Each area contains modules with a consistent internal structure. This makes the site easier to navigate and reduces cognitive load.

## Who This Course Is For

This course is suitable for learners who already have a general English level around upper-intermediate level and need stronger control in technical contexts.

Typical learner profiles include:
- engineering students;
- junior engineers;
- technicians;
- documentation contributors;
- analysts;
- project support staff;
- professionals moving from passive understanding to active technical writing.

## What the Learner Produces

The final outcome of the course is not just a set of completed exercises.

The learner progressively builds a documentation set that may include:
- glossary pages;
- language reference sheets;
- technical summaries;
- templates;
- structured exercises;
- model documentation pages;
- a final portfolio.

These materials can then be organised into a site built with **MkDocs** and versioned through **GitHub**.

## In Practical Terms

By the end of the course, the learner should be better able to:

- read technical texts more efficiently;
- describe systems, components, and processes more clearly;
- write useful technical documents in controlled English;
- communicate in a more professional and reviewable way;
- transform language practice into structured documentation output.
