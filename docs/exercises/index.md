# Exercises

## What This Section Contains

This section provides the practice layer of the course.

The exercises are designed to help learners move from:
- understanding technical language,
- to using it in controlled tasks,
- to producing clearer and more professional technical output.

The focus is not on isolated drills only, but on activities that support realistic engineering communication.

## At a Glance

The exercises in this section help learners practise:

- reading technical texts;
- strengthening vocabulary and grammar control;
- writing descriptions, emails, and reports;
- working with maintenance, troubleshooting, and safety language;
- improving oral and translation-related performance.

## How to Use the Exercises

There are three practical ways to use this section.

### 1. Sequential Practice
Work through the pages in order if you want a structured progression.

This is the best option for learners following the course step by step.

### 2. Skill-Based Practice
Choose exercises by competence if you need support in one specific area.

For example:
- use [Reading Exercises](reading-exercises.md) for technical-text comprehension;
- use [Vocabulary Exercises](vocabulary-exercises.md) and [Grammar Exercises](grammar-exercises.md) for language control;
- use [Writing Exercises](writing-exercises.md), [Report Writing Exercises](report-writing-exercises.md), and [Email Writing Exercises](email-writing-exercises.md) for written production.

### 3. Output-Oriented Practice
Use the tasks to generate reusable content for the documentation project.

Strong exercise outputs can later become:
- glossary entries;
- report models;
- documentation pages;
- structured notes;
- project assets.

## Recommended Learning Route

A practical study sequence is:

1. start with reading, vocabulary, and grammar;
2. continue with technical description and general writing;
3. move to reports and technical emails;
4. add maintenance, troubleshooting, safety, and translation tasks;
5. use oral communication tasks for final consolidation.

## Simple Working Method

For each exercise:

1. read the task carefully;
2. identify the communication purpose;
3. extract key terminology;
4. draft the answer;
5. review the wording and structure;
6. keep strong outputs for later reuse.

## Review Questions

Before considering an exercise complete, check the following:

- Is the terminology appropriate?
- Is the grammar controlled enough for a professional context?
- Is the wording clear and precise?
- Is the structure easy to follow?
- Does the output resemble a real technical document or communication task?

## Level and Difficulty

Most tasks are aligned with a **B2 professional target**.

This means the learner is expected to:
- understand technical content of moderate complexity;
- produce structured and coherent English;
- distinguish between general and technical phrasing;
- revise output for clarity and usefulness.

Some tasks are slightly denser in content. In those cases, the goal is not perfect language, but effective and controlled communication.
