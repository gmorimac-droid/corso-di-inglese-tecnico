# Scoring Notes

!!! note "Teaching use note"
    These notes are intended to stabilise correction behaviour across tasks, modules, and repository outputs. Use them to keep assessment principles explicit and repeatable.

## Premium scoring dimensions

### 1. Terminological control
Can the learner select and maintain the correct engineering vocabulary?

### 2. Documentary function
Does the text clearly behave as description, instruction, report, note, summary, or requirement set?

### 3. Evidence discipline
Are claims proportionate to the facts presented?

### 4. Structural quality
Are headings, paragraphs, and lists used effectively?

### 5. Professional tone
Does the text avoid both casual language and inflated pseudo-technical style?

### 6. Repository readiness
Could the output be inserted into the final MkDocs site after limited revision?

## Suggested interpretation scale

- **Excellent:** operationally usable with minor editorial refinement.
- **Strong:** clear, disciplined, and mostly review-ready.
- **Adequate:** understandable and broadly correct, but still classroom-like.
- **Developing:** language control exists, but documentary quality remains inconsistent.
- **Weak:** meaning is partially present, but structure and terminology are not yet reliable.
