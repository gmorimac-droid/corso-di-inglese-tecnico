# Correction Principles

!!! note "Teaching use note"
    These notes are intended to stabilise correction behaviour across tasks, modules, and repository outputs. Use them to keep assessment principles explicit and repeatable.

## 1. Correct for function before elegance

A premium correction pass asks first whether the text does its job. If the sentence still fails to define scope, status, risk, action, or evidence, lexical refinement alone is not enough.

## 2. Separate observation from conclusion

This is one of the most important habits in engineering English. Students should learn to signal whether a sentence reports:

- an observation;
- a test or check;
- an interpretation;
- a conclusion;
- a recommendation.

## 3. Reward stable terminology

Students often try to vary vocabulary too much because they associate repetition with weak style. In technical writing, stable repetition is often a sign of maturity.

## 4. Penalise unsupported certainty

Claims such as *confirmed*, *fully compliant*, *resolved*, or *safe* should be used only when the evidence boundary truly supports them.

## 5. Correct paragraph logic, not only local errors

When a paragraph lacks purpose or sequence, fixing articles and prepositions will not solve the main documentary weakness.

## Practical correction sequence

1. Identify text function.
2. Mark unstable or vague terminology.
3. Mark evidence-level problems.
4. Review sentence economy.
5. Review formatting and heading logic.
6. Suggest repository-ready revision.
