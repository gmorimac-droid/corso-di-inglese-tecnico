# Glossary Governance

## Purpose

Glossary governance ensures that terminology is controlled across the repository. In a technical English course with engineering orientation, glossaries are not supplementary vocabulary lists. They are the reference layer that stabilizes meaning, usage, register, and preferred phrasing.

Without glossary governance, the repository risks inconsistency, redundancy, weak paraphrase habits, and confusion between near-synonyms.

## Governance Objectives

The glossary system should:

- provide stable definitions;
- distinguish preferred terms from approximate or learner-generated alternatives;
- capture usage notes relevant to engineering contexts;
- support module pages and exercises;
- reduce terminological drift across contributors and revisions.

## Preferred Entry Logic

A strong glossary entry should ideally include:

- the term;
- a concise definition;
- one or more professional example sentences;
- usage notes where confusion is likely;
- related terms or contrasts where useful.

## Repository Rules

### Rule 1 — Use the glossary as reference, not as decoration
If a term is important enough to appear repeatedly, it should either already exist in the glossary system or be added there with controlled wording.

### Rule 2 — Avoid unstable synonym cycling
Do not rotate between multiple near-equivalent words merely for stylistic variation. If *fault*, *issue*, *malfunction*, and *failure* are used, each should retain its intended nuance.

### Rule 3 — Align modules with glossary language
Where possible, modules should reuse the preferred definitions and usage patterns found in the glossary pages.

### Rule 4 — Revise rather than multiply
If an existing entry is weak, improve it. Avoid creating duplicate or overlapping entries without strong reason.

## Review Questions

- Is the term defined clearly enough for B2 engineering learners?
- Is the usage context explicit?
- Is the term consistent with the rest of the repository?
- Would a learner know when to use this term and when not to use it?
- Does the entry reduce ambiguity rather than create it?

## Governance Outcome

The glossary layer should function as the linguistic backbone of the repository. Pages may vary in subject matter, but the terminology system should remain disciplined, cumulative, and reusable.
