# Technical English for Engineering B2

## A documentation-focused course for civil, industrial, dual-use, and defence-oriented engineering contexts

This site supports the development of **B2-level technical English** for engineering communication, documentation, reporting, maintenance, safety, and project work.

The course is designed not only to improve language accuracy, but also to help learners produce content that is clear, structured, and usable in professional environments.

## At a Glance

- **Level:** B2
- **Focus:** technical and professional English
- **Contexts:** civil, industrial, dual-use, and defence-oriented engineering
- **Output:** structured documentation developed with **MkDocs** and managed through **GitHub**

## What You Will Learn

By the end of the course, the learner should be able to:

- read technical texts with better speed, control, and accuracy;
- describe components, systems, interfaces, and processes in professional English;
- write effective emails, summaries, reports, requirements, and safety-related statements;
- distinguish between general English and engineering-oriented technical English;
- adapt language to different professional settings while maintaining clarity and neutrality;
- organise technical content into a structured MkDocs documentation site.

## Recommended Study Path

A practical sequence is:

1. read the pages in **Course**;
2. study the **Modules** in order;
3. use **Glossaries** and **Language Tools** as reference support;
4. complete the **Exercises**;
5. reuse the **Templates and Models** for written production;
6. develop your documentation in the **Final Project** section.

## How This Site Works

The site follows a simple learning cycle:

- **understand** a language pattern;
- **analyse** how it is used in technical communication;
- **apply** it in a task;
- **revise** the result;
- **publish** the improved version as structured documentation.

This means the site is both a course and a model of technical documentation practice.

## Suggested Entry Points

## Final Note

The course works best when learners treat it as a **documentation lab**, not only as a set of lessons.

The strongest results usually come from drafting, revising, and reusing content until it becomes clear, consistent, and publication-ready.
