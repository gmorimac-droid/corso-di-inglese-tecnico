# Learning Outcomes

## What the Learner Should Be Able to Do

By the end of the course, the learner should demonstrate a **B2-level operational command of technical English** in engineering contexts.

The target is not perfect fluency. The target is the ability to produce English that is:
- clear;
- credible;
- reviewable;
- professionally usable.

## At a Glance

By the end of the course, the learner should be able to:

- understand technical texts more effectively;
- use engineering vocabulary with better control;
- write clearer technical documents;
- communicate more accurately in professional settings;
- adapt language to different engineering environments;
- build structured documentation in MkDocs.

## Language Outcomes

The learner should be able to:

- understand technical prose, lists, tables, and document structures with improved precision;
- infer meaning from context when unfamiliar terminology appears in a recognisable engineering setting;
- distinguish essential information from supporting detail in reports, manuals, and procedures;
- identify whether a statement expresses fact, requirement, recommendation, limitation, or interpretation.

## Vocabulary Outcomes

The learner should be able to:

- use a core professional vocabulary across engineering domains;
- distinguish between general words and technical terms;
- use common technical collocations more accurately;
- maintain terminological consistency within the same page or document;
- create glossary entries with definitions, examples, and usage notes.

## Grammar and Structure Outcomes

The learner should be able to:

- use passive structures appropriately in process and report writing;
- use modal verbs accurately in requirement and procedure language;
- express sequence, cause, condition, limitation, and comparison clearly;
- write paragraphs with a clear topic and controlled sentence structure;
- revise informal wording into documentary-quality phrasing.

## Professional Communication Outcomes

The learner should be able to:

- describe components, assemblies, systems, interfaces, and workflows;
- report findings, anomalies, and corrective actions in structured English;
- write technical emails that are concise, polite, and operationally useful;
- contribute to meeting notes, action tracking, and status updates;
- present technical content in a controlled oral format such as a briefing or review intervention.

## Domain-Specific Outcomes

The learner should be able to:

- understand and use language related to maintenance, troubleshooting, safety, testing, and compliance;
- adapt wording to civil, industrial, and defence-oriented documentation contexts;
- recognise differences in tone across educational, commercial, procedural, and institutional texts;
- use neutral professional language in formal or sensitive contexts.

## Documentation Outcomes

The learner should be able to:

- convert notes and exercises into structured Markdown pages;
- organise content using headings, lists, tables, and cross-references;
- develop reusable templates, phrasebanks, and reference pages;
- contribute to a coherent MkDocs site with stable structure and version-aware logic.

## Expected Standard at Course Completion

The course does not aim at native-like fluency.

It aims at a professional standard in which the learner can produce language that is:
- sufficiently precise;
- grammatically controlled;
- well structured;
- terminologically consistent;
- adapted to audience and purpose;
- open to revision and improvement.

## In Practical Terms

A learner who completes the course successfully should be able to produce technical English that another professional can:
- read without confusion;
- review without excessive rewriting;
- reuse in documentation, coordination, or reporting work.
