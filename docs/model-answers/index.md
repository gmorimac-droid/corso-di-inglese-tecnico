# Model Answers and Guided Response Patterns

!!! note "How to read model answers"
    Model answers should be analysed, not memorised. Compare them against your own draft to identify improvements in structure, evidence handling, terminology, and documentary usefulness.

## Purpose of this section

This section provides **high-value model answers** for selected activity types used across the course. The aim is not to create memorisation material, but to show what a professionally stronger answer looks like at B2 level when the learner writes in a disciplined engineering register.

A model answer is useful only if it is read analytically. The learner should therefore examine:

- how the opening sentence defines scope;
- how terminology is stabilised across the paragraph;
- how factual information is separated from interpretation;
- how limitation and next action are expressed;
- how the closing sentence supports documentary usefulness.

## How to use model answers

The correct method is:

1. attempt the task independently;
2. compare the draft with the model answer;
3. identify specific differences in wording, structure, and logic;
4. rewrite the original answer rather than copying the model directly;
5. record the new phraseology in the learner’s own glossary or phrasebank.

## Premium learning principle

A premium course should teach not only *what a correct answer says*, but also *why that answer works under professional review conditions*. For that reason, each model answer file contains:

- a scenario;
- an answer;
- a commentary section;
- a revision note;
- a transfer note showing where the same language can be reused elsewhere in the repository.
