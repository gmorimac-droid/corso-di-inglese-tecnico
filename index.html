    Technical English for Engineering B2¶

    Professional documentation-focused course for civil, industrial, dual-use, and defence-oriented engineering contexts¶

    Repository status

    This site has been designed as a technical learning environment rather than a simple collection of lessons. The course trains the learner to use English as a working language for engineering communication, documentation, coordination, reporting, maintenance, safety, and project support. The pedagogical assumption behind the site is straightforward: in many professional settings, language quality is not ornamental. It directly affects interpretation, verification, traceability, and operational alignment.

    The course targets a B2 working level. In practical terms, this means the learner is expected to understand technical texts of medium complexity, participate in structured technical discussions, write disciplined engineering prose, and produce documentation that is clear enough to be reviewed, shared, archived, and reused. The site therefore combines language development with documentary discipline. Every major section is intended to strengthen not only vocabulary and grammar, but also structure, consistency, and professional usability.

    What the course is designed to achieve¶

    At the end of the course, the learner should be able to:

    • read manuals, specifications, procedures, and engineering reports with improved speed and precision;
    • describe components, assemblies, systems, interfaces, and workflows using accurate technical phrasing;
    • write concise but useful emails, summaries, meeting notes, reports, requirements, and safety-related statements;
    • distinguish between informal English, classroom English, and professional engineering English;
    • adapt language to civil, industrial, institutional, and defence-oriented settings without losing clarity or neutrality;
    • structure technical knowledge into a documentation site built with MkDocs and managed through GitHub.

    How to use this site¶

    The site is organised as a progressive framework.

    1. Start with the pages in About the Course to understand the scope, methodology, and assessment logic.
    2. Study the Modules in sequence if you want a guided path, or consult them selectively if you need support on a specific communication task.
    3. Use the Glossaries and Language Tools as working reference material during reading and writing.
    4. Complete tasks from the Exercises section and compare your production with the expectations stated in the module checklists and templates.
    5. Reuse the Templates and the Final Project section when preparing deliverables for the documentation site.

    Design logic of the learning path¶

    The site follows a repeated cycle:

    • Understand a technical language pattern;
    • Analyse how it behaves in real engineering discourse;
    • Apply it in a controlled exercise;
    • Revise the output according to professional criteria;
    • Publish the refined result as structured Markdown content.

    This means that the site itself is also a model. It demonstrates the type of organisation, heading hierarchy, terminological stability, and documentary clarity that the learner is expected to produce.

    Recommended navigation path¶

    A practical progression is the following:

    • Module 01–04 for foundations, vocabulary discipline, grammar, and technical reading;
    • Module 05–08 for describing objects, systems, processes, and performance;
    • Module 09–12 for technical writing, email communication, reports, requirements, and compliance language;
    • Module 13–15 for maintenance, troubleshooting, safety, and incident-related communication;
    • Module 16–17 for defence-oriented, institutional, and programme-level language;
    • Module 18 for documentation architecture with MkDocs and GitHub.

    Final note¶

    The strongest results usually come from learners who treat the course as a documentation laboratory. Instead of completing tasks once and moving on, they revise terminology, improve internal consistency, and gradually convert exercises into reusable project assets. That approach is strongly recommended here.

    V4 premium layer¶

    This repository now includes a premium layer intended for higher-value teaching and review use. In addition to the standard course pages, the site now contains:

    • model answers showing what stronger B2 engineering-English output looks like;
    • teacher notes supporting consistent correction and delivery;
    • review playbooks for repository quality and writing quality;
    • more explicit links between modules, exercises, and templates.

    The practical effect is that the site can function in three ways at once:

    1. as a learner-facing course;
    2. as a teacher-facing correction framework;
    3. as a documentation-oriented repository model for MkDocs and GitHub.

    Recommended navigation path¶

    A strong learning path is:

    1. read the module;
    2. consult the glossary or phrasebank;
    3. complete the exercise;
    4. compare with the model answer;
    5. revise the output using the review playbooks;
    6. integrate the revised version into the final repository.
